What is a Class?

So by now you may be wondering what on earth a class is. And it is a very important question to ask. In object oriented programming classes are critical to the creation of objects.

The easiest way to think of classes is as a blueprint. In the same way an architect designs a blueprint for a house and then is able to use that same blueprint over and over again to create new houses, a class can be designed once and then we can create as many objects as we want from it.

Classes contain two very important things: the data that an object needs, and the methods that it will use. For instance, a class for a person would almost certainly contain a name, age, weight and possibly also have areas for things like hair colour or a telephone number. But it would also possibly have methods for running or talking or any number of things that person object may be required to do.

What is a method?

This leads us to a very critical piece of information. What is a method? A method is any action an object may be required to perform to fulfill its desired task. Two of the most common methods you will use are Console.WriteLine() and Console.Read(). They are used to read and write information to the console. As you can see we write "Console" before each method, that's because both WriteLine() and Read() are methods that belong to the Console class.

What's with the brackets?

The brackets at the end of a method are where parameters go. Parameters are required data that the method uses to perform it's stated function. Going back to Console.WriteLine() when used in practice it actually requires a String. If you look In some of the examples in which the method is used, there is always a bunch of text in between quotation marks. This is a string being created specifically for the method which it will then take and write to the console!

We often put Hello World as the example, mostly out of tradition but you can write whatever you want. You can also take a string object you created somewhere else and put it in the parameter list. Try it out!
